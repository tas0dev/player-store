# Store Mod 要件定義

## 1. 概要

本MODは、Minecraft Fabric 1.20.1向けのプレイヤーショップMODである。

プレイヤーは専用ブロック「店テーブル」をクラフト・設置し、そのテーブルに商品と価格を設定することで、他プレイヤーにアイテムを販売できる。

GUIは使用せず、右クリック操作と名前付きアイテムによって設定を行う。

## 2. 対象環境

```text
Minecraft: 1.20.1
Mod Loader: Fabric
言語: Java
```

## 3. 目的

本MODの目的は、MMO風サーバーにおいて、プレイヤー同士が自然にアイテムを売買できる仕組みを提供することである。

単なるチェスト売買ではなく、専用の店テーブルを設置することで、見た目にも分かりやすいプレイヤーショップを実現する。

## 4. 基本仕様

プレイヤーは「店テーブル」をクラフトできる。

店テーブルはブロックとして設置できる。

店テーブルを設置したプレイヤーを、その店の店主として記録する。

店主は、店テーブルに商品と価格を設定できる。

他プレイヤーは、店テーブルを右クリックして商品を購入できる。

価格単位は「シルバー」とする。

## 5. 店テーブル仕様

店テーブルは専用アイテムから設置されるブロックである。

店テーブルはBlockEntityを持つ。

店テーブルごとに、以下の情報を保存する。

```text
ownerUuid: 店主のUUID
ownerName: 店主名
price: 商品価格
sellItem: 販売商品
stock: 在庫数
```

初期実装では、最低限 `price` の保存を実装する。

## 6. 価格設定仕様

店主は、金床で紙に数値の名前を付ける。

例：

```text
100
250
5000
```

店主がその紙を持って店テーブルを右クリックすると、紙の名前が価格として読み取られる。

正常に読み取れた場合、店テーブルのBlockEntity NBTに価格を保存する。

紙は消費されない

成功時には以下のようなメッセージを表示する。

```text
価格を 100 シルバーに設定しました
```

## 7. 価格入力制約

価格として使える文字列は、正の整数のみとする。

有効例：

```text
100
500
5000
```

無効例：

```text
0
-100
100シルバー
abc
空文字
```

無効な場合は価格を保存せず、紙も消費しない。

表示メッセージ例：

```text
数字のみ入力してください
```

## 8. 商品設定仕様

店主が販売したいアイテムを持って店テーブルを右クリックすると、そのアイテムを販売商品として登録できる。

商品登録時には、手持ちアイテムのコピーを `sellItem` として保存する。

将来的には、在庫投入と商品登録を分ける。

初期実装では、商品設定は後回しにし、価格設定を優先する。

また、テーブルの上にアイテムを表示（額縁のように）する。

## 9. 購入仕様

他プレイヤーが店テーブルを右クリックすると、設定されている商品を購入する。

購入時には以下を確認する。

```text
価格が設定されている
商品が設定されている
在庫が存在する
購入者が十分なシルバーを持っている
購入者のインベントリに空きがある
```

購入が成功した場合、以下を行う。

```text
購入者のシルバーを減らす
店主のシルバーを増やす
在庫を減らす
購入者に商品を渡す
```

## 10. 権限仕様

店テーブルの設定を変更できるのは店主のみとする。

店主以外が価格設定用の紙を使って右クリックしても、価格は変更できない。

店主以外は購入操作のみ可能とする。

将来的には、店テーブルの破壊も店主のみ許可する。

## 11. 経済仕様

通貨単位は「シルバー」とする。

プレイヤーごとにシルバー残高を持つ。

将来的に以下の機能を追加する。

```text
残高確認（tabとかで表示）
支払い
売上受け取り
管理者による残高変更
```

## 12. 保存仕様

店テーブルの価格や商品情報は、BlockEntity NBTに保存する。

ワールドを閉じても価格設定が消えないことを必須とする。

最低限保存するNBTは以下。

```text
price: int
```

将来的に保存するNBT。

```text
ownerUuid: String
ownerName: String
sellItem: NBT
stock: int
```

## 13. 非機能要件

サーバー側のみで価格設定・購入処理を行う。

クライアント側だけで残高や在庫が変更されてはならない。

不正な価格、未設定商品、在庫不足、残高不足では処理を中断する。

クラッシュやリログ後も店データが保持される。

## 14. 初期実装範囲

最初に実装する範囲は以下とする。

```text
店テーブルアイテムの追加
店テーブルブロックの追加
店テーブルのクラフトレシピ追加
店テーブルのBlockEntity化
紙の名前から価格を読み取る
価格をBlockEntity NBTに保存する
価格設定成功メッセージを表示する
```

## 15. 後回しにする機能

以下は初期実装では行わない。

```text
GUI
複数商品販売
買い取り機能
オークション
依頼掲示板
店検索
税金
価格履歴
NPC店員
```

## 16. 最小完成形

最小完成形は以下である。

```
プレイヤーが店テーブルをクラフトする
店テーブルを設置する <- 店主として記録される
紙に「100」と名前を付ける
紙を持って店テーブルを右クリックする
店テーブルに価格100シルバーが保存される
再起動後も価格が保持される
```

この段階を完成させてから、商品登録と購入処理を追加する。
